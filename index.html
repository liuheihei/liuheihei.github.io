数据结构笔记

第六章 数

数的定义:数是n个节点的有限集，n=0时称为空树，在任意一颗非空数中，有且仅有一个特定的称为根的结点。当n>1时，其余结点可分为m(m>0)个互不相交的有限集T1、T2….Tm,其中每个集合本身又是一棵树并且称为根的子数。

  • n>0时，根结点是唯一的，不可能存在多个根结点。
  • m>0时，子数的个数没有限制，但是它们一定是互不相交的。

结点拥有的子树数称为结点的度。度为0的结点称为叶结点或终端结点。除根结点之外，分枝结点也称为内部结点，数的度是数内各结点的最大值。

结点间关系:Child、Parent、Sibling(兄弟)

结点的层次：树中结点最大的层次称为数的深度或者高度。

如果将书的各子树看成是从左至右有次序的，不能互换的称为有序树。否则称为无序树

森林：是m(m>0)课互不相交数的集合。

线性表和数结构的不同：

线性表：

  • 第一个数据元素：无前驱
  • 最后一个数据元素：无后继
  • 中间元素：一个前驱一个后继

数：

  • 根结点：无Parent，唯一
  • 叶结点：无Child，可以多个
  • 中间结点：一个Parent多个Chile

数的存储结构

  • Parent表示法
  • Child表示法
  • Sibling表示法

Parent表示法：每个结点中，附设一个指示器指示其Parent到链表的位置。根结点的指针域设置为-1。{数据域、指针域}
数据域：存储结点数据信息
指针域：存储该结点Parent在数组中的下标

Parent表示法的好处：可以方便的根据Child结点找到Parent，但是只能以遍历的方式查找某结点的Child

Child表示法：每个结点的Child结点排列起来，以单链表作为存储结构，即n个结点有n个Child链表，如果是叶结点则次单链表为空。n个头指针又组成一个线性表，采用顺序存储结构，存放在一个一维数组中。

Sibling表示法：任意一棵树，它的结点的第一个Child如果存在就是唯一的，它的右兄弟如果存在也是唯一的，因此可以设置两个指针，分别指向该结点的第一个Child和右兄弟。

二叉树的定义

二叉树：是n（n>=0）个结点的有限集合，该集合或者为空集（空二叉树），或者有一个根结点和两颗互不相交的、分别称为根结点的左子树和右子树的二叉树组成。

二叉树的特点：

  • 每个结点最多右两颗子树，所以二叉树中不存在度大于2的结点。
  • 左子树和右子树是有顺序的，次序不能颠倒。
  • 即使某结点只有一颗子树，也要区分是左子树还是右子树。

二叉树的基本形态：

  • 空二叉树
  • 只有一个根结点
  • 根结点只有左子树
  • 根结点只有右子树
  • 根结点既有左子树又有右子树

特殊二叉树

  • 斜数：只有左结点或者只有右结点的二叉树
  • 满二叉树：在一颗二叉树中，如果所有分支结点都存在左子树和右子树，并且所有叶子都在同一层上。
    • 特点：叶子只能出现在最下一层。
    • 非叶子结点的度一定是2
    • 同样深度的二叉树中，满二叉树的结点个数最多。叶子数最多。
  • 完全二叉树
    对于一颗具有n个结点的二叉树按照层次编号，如果编号为i(1<=i<=n)的结点与同样深度的满二叉树中编号为i的结点在二叉树的位置相同，则称为完全二叉树。
    • 完全二叉树的特点
      • 叶子结点只能出现在最下两层
      • 最下层的叶子一定集中在左部连续位置
      • 倒数二层、若有叶子结点，一定都在右部连续位置。
      • 如果结点度为1，则该结点只有左子树、不存在有右子树的情况。
      • 同样结点树的二叉树，完全二叉树的深度最小。

RHEL-Debian包管理文档

发表于 2017-01-16 | 分类于 技术 |

ubuntu 软件包使用

之前一直在RHEL系列工作，安装软件系列的操作十分熟悉了，比如rpm、yum命令都是使用的非常熟悉。但是debian的apt-get、dpkg确很生疏，今天有空记录一下RHEL下经常使用的操作，并对应Debian系列如何使用。

Debain

dpkg

安装软件

$ dpkg -i <package_name> #保留配置
$ dpkg -P <package_name> #不保留配置

卸载软件

$ dpkg -r <package_name>

查看包的相信信息

$ dpkg -s <package_name>

查询某文件是由哪个包生成的

$ dpkg -S <filename>

查询包里安装里哪些文件

$ dpkg -L <package_name>

查询某包是否被安装

$ dpkg -l <package_name>
$ dpkg -l | grep <package_name>

列出所有被安装的包

$ dpkg -l
apt-get命令

安装包

$ apt-get install <package_name>

指定安装包版本

$ apt-get install <package_name>=1.1.1

卸载软件

$ apt-get remove <package_name> #保留配置
$ apt-get remove --purge <package_name> #删除配置
$ apt-get autoremote #删除包以及其依赖的软件包
$ apt-get autoremote --purge #删除包以及其依赖的软件包并删除配置文件

删除本地缓存

$ apt-get clean
$ apt-get autoclean

更新和升级

$ apt-get upgrade <package_name>

查询某个文件哪个包存在

$ apt-file search <filename>

RHEL下软件包管理

rpm 命令

安装软件

$ rpm -ivh <package_name>

卸载软件

$ rpm -e <package_name> ( 如果有依赖关系，yum命令会更简单一点。)

查询某文件是由哪个包生成的

$ rpm -qf <filename>

查询包里安装里哪些文件

$ rpm -qf <package_name>

查询某包是否被安装

$ rpm -q <package_name>

列出所有被安装的包

$ rpm -qa

升级一个包

$ rpm -Uvh <package_name>

$ rpm -Uvh --oldpackage <package_name> #降级
yum命令

安装包

$ yum install <package_name> #自动安装依赖包

安装包组

$ yum groupinstall <group_name>

卸载软件

$ yum remove <package_name>

删除本地缓存

$ yum clean all

查询源中有哪些包

$ yum list

更新和升级

$ yum update / yum upgrade  #全部更新

$ yum update <package_name> # 更新单个包以及所依赖的包

yum update 和 yum upgrade 的区别(google 找的并未测试)

yum upgrade == yum update --obsoletes
yum upgrade  会升级包到最新到版本，举个例子，foo 新版改名为 foo2，那么它会安装 foo2
并且把 foo 删除，而 yum update 只会升级 foo，不会将包更新到 foo2。
因此yum update 可以更保守的管理包，但是yum upgrade 会保持包的状态是最新的。

查询某个文件哪个包存在

$ yum provides "*/<filename>"

Docker Harbor Registry 使用

官网介绍：

功能

1.使用Harbor管理项目
2.管理项目中的成员
3.复制到一个其他到Registry
4.搜索工程和源
5.管理Harbor系统
  管理用户
  管理副本的地址
  管理复制策略
6.使用docker client pull和push镜像
7.删除源和镜像
用户权限
项目级别
Guest：Guest指定的项目只有只读的权限
Developer：指定的项目有读写的权限
ProjectAdmin：当新创建项目的时候，应该指定一个“ProjectAdmin”用户
    该用户有读写权限，也能进行权限管理例如添加或者删除成员
系统级别
SysAdmin：拥有最高权限，可以设置一个用户作为管理员，共有的项目“library”
Anonymous: 当一个用户没有登陆的时候，没有权限访问私有项目，但是有公有项目的读权限。
用户认证：
一、数据库
  在这个模式下用户可以自注册（可以关闭）
  关闭的时候只有系统管理员可以添加用户，用户名和邮箱地址必须相同。

二、LDAP
管理项目

一个Harbor项目包含一个应用的所有的repo，在这个项目创建之前不能push镜像，RBAC应用在项目中
Harbor中有两种项目类型。

一、public
  所有的用户有读权限，当你想分享给其他人的时候有用。
二、private
  只能登陆之后才有访问权限。

所有的日志可以在“logs”按钮中列出，可以通过用户名，操作来过滤。

管理项目中的成员
添加成员
更新和删除成员
复制image

该功能被用于从一个Harbor实例复制到另一个实例。

复制的函数是“project-oriented”,一旦系统用户设置了一个policy到另一个项目，这个项目下的所有repo都会被复制到远端的registry。每一个repo都会启动一个job来运行复制，如果这个项目在远端registry不存在就会自动创建一个，如果已经存在，但是配置的用户没有写权限，就会失败，当一个新的repo在这个项目中创建，相同的操作也会在远端的registry，删除操作也一样，但是用户信息不会被复制过去。

由于网络的原因，在复制的过程中可能有延时，如果由于网络原因复制失败复制进程在几分钟之后会重试。

一个0.3.5之前的版本不能跟0.3.5之后的版本进行同步

管理员选项
管理用户
管理Destination（不理解）
管理replication
注意在修改replication的时候需要停止这个policy

如何使用 docker client 进行pull and push

Harbor只支持registry v2 所以docker client版本需要高于1.6.0

Harbor 支持HTTP，但是默认docker client会优先尝试HTTPS，如果私有registry使用HTTP或者使用一个未知CA的HTTPS，需要在docker client中添加一行 --insecure-registry myregistrydomain.com:5000如果使用HTTPS只需要简单的将证书存放在/etc/docker/certs.d/myregistrydomain.com:5000/ca.crt中

使用docker client push image 过程

1.登陆

$ docker login {{  ip  }}

2.标记image

$ docker tag ubuntu:14.04 {{  ip  }}/demo/ubuntu:14.04

3.push image

$ docker push {{  ip  }}/demo/ubuntu:14.04

2016-12-11-lvs


NAT网络地址转换

工作流程

1.client访问vip.
2.vip将tcp包中的目的地址根据分发策略改变成realserver的ip地址以及端口
    （此时包的源地址为客户端ip地址，目的地址为realserver地址如果需
    要配置iptables需要注意此时包源地址不是lvs的地址所以realserver
    中对应的端口需要开放给所有人）
    (lvs会在自己的应用池里面记录每个包所修改的ip,用于记录当这个包返回时修改为原来的状态)
3.realserver获取到tcp包之后将返回给负载均衡器。
    （出去面试的时候有被问到这个包是怎么回的负载均衡器，需要将realserver的网关指向lvs）
    （此时返回的包源地址为realserver地址，目的地址为客户端地址）
4.vip获取到返回的tcp包之后，修改这个包中的源地址。将realserver的地址转换为vip的地址。
5.客户端收到服务器发回的包。

DR模式

工作流程

1.client访问vip
2.vip将tcp包中目的mac地址修改为realserver的mac地址并在局域网内广播。
3.realerver收到广播处理包在将tcp包返回给客户端

隧道模式

工作流程

1.client访问vip
2.vip将tcp包再次封装并发送给realserver，
3.realerver收到包后打开封装，发现里面的目的ip就是自己，处理完之后直接发送给客户端。

调度算法

轮叫调度（Round-Robin）

不管服务器当前的连接数和相应速度，所有请求平均分配

加权轮叫调度（weighted Round-Robin）

根据对应的权重表示服务器的处理性能，权重高的优先处理更多的请求

最小链接数调度（Least-connection）

把新请求分配到当前连接数最小的服务器，它通过服务器当前活跃的连接数来估计服务器的负载情况。

加权最小链接调度（Weighted Least-Connection）

把新请求分配给 当前连接数除以权重值最小的服务器，

基于局部的最少链接调度（Locality-Based Least Connections ）

根据目标ip地址找出目标ip地址最近使用过的服务器，
若1.该服务器存活. 
2该服务器没有超载并且没有其他服务器负载小于该服务器二分之一
则选择该服务器，否则选择一个新的服务器处理该请求

第二课 函数

python中的def语句实际上是一个可执行语句：当它运行时创建一个新的函数对象并将其赋值给一个变量名（函数名）。

def只有在运行时才会进行评估，而在def之中的代码在函数调用时才会评估。也就是当函数没有被调用时，函数中的代码将被忽略。

问题：下列关于函数的写法中，各有什么区别？

def test_fun():
    print("test")

test_fun
test_fun()
print(test_fun)
print(test_fun())

答：在python中函数也是一个对象，如果不加括号，指的是函数本身，而不会调用函数。test_fun()才是调用函数，所以print（test_fun）打印出来的是。而print（test_fun()）最终为什么有一个None，是因为函数没有返回值，第一个打印出来的test是由test_fun()打印出来，而print打印的应该是函数的返回值，该函数没有返回值，所以打印为none。

第三课 python对象

什么是对象？

python中一切皆对象，比如坐着的椅子是一个对象，玩的电脑是一个对象。通过描述属性（特征）和行为来描述一个对象，比如正在摇尾巴的小狗，是它的行为，他的年龄，体重是它的属性，

在描述对象的时候分为两个方面：
    1.它可以做什么（行为，在python中称之为方法）
    2.它是什么样的（属性）

所有的python对象都有三个特性：身份、类型、值

身份：每个对象都有唯一的身份标示自己，任何对象都可以通过内建函数id()来得到身份。
类型：决定了对象可以保存什么类型的值，可以进行什么样的操作，以及遵循什么样的规则使用type()来查看对象的类型。
值：对象表示的数据项。
