Docker Harbor Registry 使用

官网介绍：

功能

1.使用Harbor管理项目
2.管理项目中的成员
3.复制到一个其他到Registry
4.搜索工程和源
5.管理Harbor系统
  管理用户
  管理副本的地址
  管理复制策略
6.使用docker client pull和push镜像
7.删除源和镜像
用户权限
项目级别
Guest：Guest指定的项目只有只读的权限
Developer：指定的项目有读写的权限
ProjectAdmin：当新创建项目的时候，应该指定一个“ProjectAdmin”用户
    该用户有读写权限，也能进行权限管理例如添加或者删除成员
系统级别
SysAdmin：拥有最高权限，可以设置一个用户作为管理员，共有的项目“library”
Anonymous: 当一个用户没有登陆的时候，没有权限访问私有项目，但是有公有项目的读权限。
用户认证：
一、数据库
  在这个模式下用户可以自注册（可以关闭）
  关闭的时候只有系统管理员可以添加用户，用户名和邮箱地址必须相同。

二、LDAP
管理项目

一个Harbor项目包含一个应用的所有的repo，在这个项目创建之前不能push镜像，RBAC应用在项目中
Harbor中有两种项目类型。

一、public
  所有的用户有读权限，当你想分享给其他人的时候有用。
二、private
  只能登陆之后才有访问权限。

所有的日志可以在“logs”按钮中列出，可以通过用户名，操作来过滤。

管理项目中的成员
添加成员
更新和删除成员
复制image

该功能被用于从一个Harbor实例复制到另一个实例。

复制的函数是“project-oriented”,一旦系统用户设置了一个policy到另一个项目，这个项目下的所有repo都会被复制到远端的registry。每一个repo都会启动一个job来运行复制，如果这个项目在远端registry不存在就会自动创建一个，如果已经存在，但是配置的用户没有写权限，就会失败，当一个新的repo在这个项目中创建，相同的操作也会在远端的registry，删除操作也一样，但是用户信息不会被复制过去。

由于网络的原因，在复制的过程中可能有延时，如果由于网络原因复制失败复制进程在几分钟之后会重试。

一个0.3.5之前的版本不能跟0.3.5之后的版本进行同步

管理员选项
管理用户
管理Destination（不理解）
管理replication
注意在修改replication的时候需要停止这个policy

如何使用 docker client 进行pull and push

Harbor只支持registry v2 所以docker client版本需要高于1.6.0

Harbor 支持HTTP，但是默认docker client会优先尝试HTTPS，如果私有registry使用HTTP或者使用一个未知CA的HTTPS，需要在docker client中添加一行 --insecure-registry myregistrydomain.com:5000如果使用HTTPS只需要简单的将证书存放在/etc/docker/certs.d/myregistrydomain.com:5000/ca.crt中

使用docker client push image 过程

1.登陆

$ docker login {{  ip  }}

2.标记image

$ docker tag ubuntu:14.04 {{  ip  }}/demo/ubuntu:14.04

3.push image

$ docker push {{  ip  }}/demo/ubuntu:14.04

Harbor 安装文档

下载在线安装包

https://github.com/vmware/harbor/releases

修改配置harbor.cfg

1.hostname (可以是IP或者FQDN)
2.ui_url_protocol (HTTP or HTTPS)
3.email settings (用于发送用户重置密码邮件)
4.Harbor_admin_password (管理员登陆密码)
5.auth_mode (设置为mysql或者LDAP)
6.db_password (mysql password)
7.self_registration (自注册yes or no)
8.max_job_workers (最大replication job数量)
9.token_expiration (token过期时间)
10.verify_remote_cert (跳过证书检查，适用于自签名证书)
11.customize_crt (自己生成证书)

harbor.cfg

hostname = sredev2.test.com
ui_url_protocol = https
email_identity =
email_server = smtp.sohu.com
email_server_port = 25
email_username = liuheihei@sohu.com
email_password = xxxxx
email_from = liuheihei@sohu.com
email_ssl = false
harbor_admin_password = xxxxxxxx
auth_mode = db_auth
ldap_url = ldaps://ldap.mydomain.com
ldap_basedn = ou=people,dc=mydomain,dc=com
ldap_uid = uid
ldap_scope = 3
db_password = xxxxxxx
self_registration = off
use_compressed_js = on
max_job_workers = 3
token_expiration = 30
verify_remote_cert = on
customize_crt = off
crt_country = CN
crt_state = State
crt_location = CN
crt_organization = organization
crt_organizationalunit = organizational unit
crt_commonname = example.com
crt_email = example@example.com
project_creation_restriction = everyone
ssl_cert = /data/cert/sredev2.test.com.crt
ssl_cert_key = /data/cert/sredev2.test.com.key

数据默认存储在本地文件系统，如果要修改，配置文件在common/templates/registry/config.yml

安装需要的docker环境

安装docker

https://docs.docker.com/engine/installation/linux/debian/

安装docker-compose

默认会生成docker-compose
使用docker-compose来管理Harbor
如果要改变Harbor的配置，首先停掉Harbor实例，然后更新harbor.cfg，最后运行install.sh

删除Harbor

删除Harbor容器但是保存数据

$ docker-compose down

删除数据 

$ rm -r /data/database
$ rm -r /data/registry

持久化数据、日志
Harbor使用rsyslog来保存日志，日志文件被存放在/var/log/harbor
默认registry数据持久化在主机的/data目录下，当Harbor被删除这个数据依然被保存。

配置https

自签名证书

$ openssl req -newkey rsa:4096 -nodes -sha256 -keyout ca.key -x509 -days 365 -out ca.crt

$ openssl req -newkey rsa:4096 -nodes -sha256 -keyout sredev2.test.com.key -out sredev2.test.com.csr

$ openssl x509 -req -days 365 -in sredev2.test.com.csr -CA ca.crt -CAkey ca.key -CAcreateserial -out sredev2.test.com.crt

客户端添加信任(自签名证书会报错)

Linux
    $ mkdir /etc/docker/certs.d/sredev2.test.com -p
    $ cp sredev2.test.com.crt /usr/local/share/ca-certificates/sredev2.test.com.crt
    $ update-ca-certificates

MAC 端
将IP：port添加到preferences --> advenced --> inseure registries

docker login 失败：需要修改下面参数 

$ sudo sysctl -w net.ipv4.tcp_mtu_probing=1

修改sysctl.conf net.ipv4.tcp_mtu_probing=1
